# AI 讲题业务商业逻辑与战略推演 BP

## 零、 核心哲学基调：“某事 + AI”，而不是“AI + 某事”

**这是本战略的最核心共识：我们绝不是在拿着大模型这把锤子到处找教育的钉子（AI + 教育），而是在给一个原本就跑得通、能赚钱的传统机构业务加杠杆（教育/加盟业务 + AI）。**

不要用纯技术极客的视角去打造产品，必须从“试图用 AI 替代活人老师”的傲慢做根本性转向。团队唯一正确的打法：**拿着原本引以为傲的“视频渲染技术”，去做那些大机构老板眼里最粗暴的“外包 API 插件 / 降本增效印钞机”。**

---

## 一、 认知重构：打破团队的“技术傲慢与教研执念”

如果要跑通这个业务，技术团队必须先承认自身的短板，并破除对“AI能颠覆教育”的幻想。

1. **单纯 To-C 讲题碰不得的四大死穴**：
   - **拿不到燃料**：学生的错题和行为数据等核心数字资产被巨头生态死死垄断。
   - **基因不对**：团队长板在“视频渲染”，但教育的深水区是枯燥的教研和题型拆解。
   - **无法自证**：教育要为结果（提分）买单，家长不会为没立竿见影效果的“AI 播放器”付大钱。
   - **忽视教育本质**：AI 可以完美地“教”，但对付人类的惰性，需要活人老师的“监督眼光与情绪陪伴（育）”。

2. **“做绝对提分的有效内容”是一个彻头彻尾的伪命题**：
   - 没有十年沉淀的教研基因，试图靠大模型自动生成一段视频就让学生“成绩飞升”，是极其傲慢的。
   - 教育提分的真相，是靠线下机构用规矩、死磕、陪伴和海量刷题“逼”出来的。**技术团队绝不能陷入“我要用 AI 对学生成绩负责”的道德自嗨与教研泥潭中。**

---

## 二、 商业模式重塑（从“卖软件”到“卖获客核武器”）

一旦承认了“真实提分”是伪命题而且是线下机构的活儿，我们的商业模式就清晰了——我们要造的是一把**极其锋利的卖课铲子**。

### 1. 终极奥义：B 端想要的是“成品结果（RaaS）”，而不是“工具（SaaS）”
老板极其讨厌引入增加了学习成本的“工具后台”。我们不要卖给机构一个“需要他们自己填入教研大纲”的复杂排版软件。我们需要交付**黑盒化的结果**：
客户只管通过 API 或微信传入一道题的图片，我们直接吐出一个讲解完美的 MP4/H5 课件成品。客户按“生成的成品条数”买单，而不是按“软件年费”吃灰。

### 2. 不碰教学，只做“面向家长的极客获客道具”
老板买我们的系统，根本不是为了给学生做教学工具，而是为了买一套用于锁客和涨价的“重型包装纸”。
* **制造“视觉震撼”，化身售前体验大杀器**：家长根本没有专业能力去判断一道题讲得好坏。当他们在体验课后，看到机构用 3 分钟极速生成了冠以孩子名字的极度炫酷、科技感拉满的动态黑板剖析视频时，家长会直觉认为“这极其有效”，从而痛快掏钱。
* **高客单价的提价借口**：专属 AI 错题视频能帮机构从普通的“同质化大班课”，包装成前沿科技加持的“SVIP 伴学服务”，直接拉升利润。教学脏活留给老师，我们只负责让机构显得“极其高级”。

### 3. 顺手击穿机构的两大基建痛点
* **异地扩张的难题**：新分校最怕没有好老师“镇场子”。我们的系统是“名师数字印刷机”，当地员工只需播放高保真视频，跨城复制的试错成本降到最低。
* **新老师培养成本**：直接**剥夺应届老师讲题的技术门槛**，把要求降维成“播报员和心理辅导员”，彻底打下人力成本。

## 三、 产品形态与技术管线（不懂教研怎么做引擎？）

没有教研能力，又要交付完美的商业成品，我们的技术底线和交付路径如下：

1. **做“视觉排版引擎”，把教研甩锅给客户**
   我们绝不做“解题大脑”，只做“数字印刷机”。让机构输入“标准答案”或“老师写好的大纲”，我们的引擎只负责提取结构，并将其惊艳地转化为带板书、动画的高表现力视频。**所谓“自带食材，我只提供高端加工”。**

2. **用 Multi-Agent 工作流将“教研”工程化**（吞噬摩擦力）
   在内部用代码构建**“AI 虚拟教研流水线”**。拆分为专职 Agent：`解题大脑`（算答案） -> `教研大脑`（切分为引出考点、拆步推导、防坑总结的脚本） -> `编导大脑`（音视频渲染匹配）。用极客工程管线逼近老教师的直觉。
   *在此黑盒内部，我们用廉价大学生人工进行最后一秒的高效兜底，把“与大模型斗智斗勇的摩擦力”自己咽下去，端给客户极致爽感的零门槛结果。*

3. **内容双轨制与交互形态降维打击**
   - **双轨制**：第一套是战术级“讲题视频”（针对具体题目拆解）；第二套是战略级“讲知识点视频”（串联例题）。两者算力与定价不同。
   - **交互式 H5**：最好的交付不是单向播放的死视频，而是随时暂停、点击作答、分步引导的交互式 H5。这种“点读机”属性的外壳能带来远超裸视频的溢价。

## 四、 商业护城河与灵魂防坑拷问

发车之前，必须用这五个致命问题拷问自己：

1. **为什么是中小团队做，教培巨头（如好未来）不能自己做？** 
   - **中立的“军火商”生态位（核心护城河）**：全天下的机构都不会把命脉放在竞争对手开发的系统上。他们只敢买我们这种**“绝对中立、永远不下场招学生”**的纯技术团队的武器。
   - **基因壁垒**：教育公司的基因是“堆人重教研”，跨界去搞多并发视频渲染和多模态 Agent，极易烂尾。
2. **谁去喝大酒？** 这是一个极其纯正的 To-B 卖插件方案。如果没有前培训圈出身、手里捏着老总大把微信并能地推拿单的合伙人，纯技术宅会饿死在第一公里。
3. **致命的运营基因：我们能搞定“人机结合的富士康”吗？** 既然做 RaaS 交付完全准确的成品，面对当下 AI 的幻觉，内部必定需要建立“人工审核兜底”的流水线环节。作为一个纯技术团队，我们是否具备招募、管理几十上百个兼职大学生这种“低端密集型人力运营”的耐心与基因？被重度运营拖垮的风险必须提前做预案。
4. **防白嫖的锁装了没？** 如果只生成 mp4，老板几百块狂刷一千条就退订了。必须把分享动作绑定成我们专属且扣费的“前端带 Logo 的播放组件”。
5. **小鹅通这种大厂免费了我们咋办？** 我们的生路是对特定的考试教研表现形态（如 A-level 极具逼格的独特纯动画逻辑）建立极速适配和模板积累，做“精装定制外包”，而非廉价接口。
6. **远期怎么赚大钱（SaaS 特洛伊木马）？** 前期绝不自己搞底层模型或教研。先用低价/免费的生成工具作为诱饵切入B端，真正的宏大叙事隐藏在后端——通过各机构工具聚合，反向建立底层学情数据网，这才是未来真正值钱的壁垒。
