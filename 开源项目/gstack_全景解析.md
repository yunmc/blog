# gstack：大模型时代的高阶 AI 编程工作流框架

## 1. 项目简介
**gstack** 是由 Y Combinator (YC) 总裁 **Garry Tan** 于 2026 年 3 月开源的一套专门针对 **Claude Code**（Anthropic 的 CLI 代码助手）构建的结构化工作流框架和技能包（Skill Pack）。

在 AI 编程（Vibe Coding）时代，单兵作战的程序员面临的最大问题往往不再是如何写出某段代码，而是**如何像团队一样进行架构设计、需求把控和质量保障**。gstack 的核心理念正是将硅谷顶尖的工程管理经验固化为 AI 的提示词和工作流，将单个 AI 助手转化为一支配置齐全的**“虚拟研发团队”**。

## 2. 核心特性
- **虚拟角色扮演 (Agent Personas)**：框架内置了约 18 种不同的专家角色预设和 23 个专用工具。AI 不再是一个没有立场的代码补全器，而是可以随时切换为 CEO、产品经理（PM）、系统架构师、前端工程师或 QA 测试员等身份。
- **强制研发管线 (Enforced SDLC)**：拒绝“上来就写代码”。gstack 强制推行标准的软件开发生命周期：从产品愿景推演 ➡️ PRD 撰写 ➡️ 架构评审 ➡️ 编码实现 ➡️ 自动化测试 ➡️ 最终发版。
- **基于 Slash Command 的快捷指令**：无缝集成于 Claude Code，开发者可以通过简单的单行命令（例如 `/qa` 或 `/release`）来驱动整个自动化研发流水线。

## 3. 适合的项目与场景

**非常适合的项目类型（Sweet Spots）：**
1. **MVP（最小可行性产品）/敏捷原型开发**：创业团队需要在几天内快速把想法变成一段可运行的 Demo 以验证市场。使用 `/ceo`、`/pm` 能极快地把发散的想法收敛成具体的 PRD 和执行步骤。
2. **从零开始的新项目（Greenfield Projects）**：这类项目没有历史代码包袱。AI 可以一开始通过 `/architect` 规划出清晰、现代化的目录和架构，避免日后频繁重构。
3. **中小型全栈 Web / 移动端应用**：例如基于 Next.js、React、Vue 或 Flutter 等主流现代框架。大模型在这类拥有海量开源经验的技术栈上有着极高准度。
4. **内部工具与后台管理系统（Admin Panels）**：此类应用往往需求明确且结构固定（表单、列表、CRUD），是 AI 开发“舒适区”，能借助 `/dev` 与 `/qa` 高效产出且保证单元测试覆盖率。
5. **补齐短板的跨界全栈项目**：如果你是一名前端工程师，但突然遇到需要编写带认证系统和数据库规划的后端服务项目，gstack 的后端/DBA 角色正好可以帮你补足技术拼图。

**不太适合的项目类型：**
- **庞老且错综复杂的遗留系统（Legacy Codebases）**：如果是十年历史、百万行级别的屎山代码，极其容易超出 LLM 能够有效摄取的上下文。过多的黑盒逻辑会让 AI 架构师误判。
- **高精尖或极其底层的硬核研发**：如自研数据库内核、核心加密算法、硬件驱动开发。这些领域的工程门槛高、容错低，AI 容易产生强迷惑性的幻觉。
- **核心金融交易与结算系统**：要求 100% 严谨且不容任何差错，AI 自动部署存在风险。

## 4. 如何安装与使用

### 4.1 前置准备
由于 gstack 是深度绑定 Claude Code 的，你需要先安装并登录 Claude Code CLI：
```bash
npm install -g @anthropic-ai/claude-code
claude login
```

### 4.2 安装 gstack
通常通过克隆 gstack 的仓库，并将其配置文件（例如特定的 `.prompt.md`、`.instructions.md` 或技能包目录）引入到你的项目工作区中。
```bash
git clone https://github.com/garrytan/gstack.git
cd your-project-dir
# 将 gstack 规则链入项目
cp -r ../gstack/.claude ./ 
```
*(注：具体安装步骤以当时官方 Github Repo 的最新 README 为准)*

### 4.3 核心流转指令示例
在 Claude Code 的交互式命令行中，你可以直接使用 gstack 预设的 Slash 指令来推进项目：

1. **构思阶段：**
   ```bash
   > /ceo 我想做一款针对程序开发的番茄钟应用，带有 AI 情绪安抚功能。
   ```
   *CEO 角色会帮你分析市场契合度、核心痛点并给出产品方向。*

2. **产品与设计阶段：**
   ```bash
   > /pm 根据 CEO 的愿景生成 PRD 文档。
   ```
   *PM 角色会生成详细的需求文档和验收标准。*

3. **架构阶段：**
   ```bash
   > /architect 审查当前的 PRD，并给出技术栈选型与系统架构图。
   ```
   *架构师角色会规避技术债，规划项目目录。*

4. **开发阶段：**
   ```bash
   > /dev 开始实现核心的番茄钟计时器模块。
   ```

5. **质量保障与发布：**
   ```bash
   > /qa 对刚刚生成的计时器模块编写单元测试，并执行边界用例测试。
   > /release 打包当前版本并生成 Release Notes。
   ```

## 5. 总结
gstack 的出现标志着 AI 编程从“Copilot（副驾驶）”模式向“虚拟外包团队”模式的跃升。它不提供更聪明的基础模型，而是提供了**更聪明的协同规范**。对于希望在 AI 时代实现十倍杠杆的开发者而言，gstack 是一个极具启发性的工程实践框架。
