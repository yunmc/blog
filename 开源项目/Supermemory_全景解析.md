# Supermemory 全景解析：给 AI 赋予长期记忆

## 1. 项目简介
Supermemory 是一个旨在解决大模型“金鱼记忆”问题的开源项目，主要提供**上下文管理架构**。它通过为日常使用的 AI 客户端（如 Cursor、VS Code、Claude Desktop 等）接入持久化的记忆库，让 AI 在跨越多次对话时依然能保留用户的偏好、事实设定和项目背景。

## 2. 核心功能及应用场景表现
- **自动保存与遗忘（Memory 工具）**：从日常交流中智能抽取有价值的内容（如技术栈、业务痛点），过滤无用的日常闲聊。
- **主动回忆与检索（Recall 工具）**：AI 在回答前主动查询相关过往信息进行参考。
- **注入上下文（Context 工具）**：通过命令（如 `/context`）在对话伊始将用户画像（Preferences, recent activities）直接注入系统中。

**场景示例**：
第一天在此项目中告诉 AI：“我当前项目只用 TypeScript 和 Tailwind。” 第二天重启软件开启新对话，直接让其帮忙“写一个按钮组件”，AI 就会依靠主动回忆记忆库，直接输出 TS + Tailwind 的代码，无需你再次重复前置要求。

## 3. 核心机制与实现原理
Supermemory 并非直接将所有聊天记录作为上下文，而是结合了**大模型工具调用（Tool Calling / MCP）**与**高级检索增强生成（Advanced RAG + 信息抽取）**技术：

### 3.1 入口层（工具调用能力）
依托各大模型框架的 Function Calling 和通用的 MCP（模型上下文协议）：
- **触发存储**：大模型发现有价值的新事实时，在后台推理暗中调用 `addMemory`。
- **触发读取**：大模型判断回答需要参考前置习惯时，主动调用 `searchMemories`。

### 3.2 存储与处理层（事实抽取引擎）
- **知识提取（Fact Extraction）**：不再粗暴地塞入大段聊天记录（降噪），而是将文本提炼成具体、可独立表达的“事实（Facts）”。
- **基于容器的沙箱隔离（Container Tags）**：通过 `containerTag`（如 `tag: project-A`, `tag: personal`）将工作与私人的记忆完全隔离开，确保隐私和不相关业务上下文互不交叉。

### 3.3 检索架构（混合检索 Hybrid Search）
检索系统使用的是 RAG + Memory 的混合流：
- **向量检索（Semantic Search）**：通过语义匹配当前问题相关度最高的记忆碎片。
- **用户画像（User Profiles）**：系统能在毫秒级延迟下，把相同标签下的稳定事实合并，直接打包作为 System Prompt 提供增强指导。

### 3.4 记忆净化与进化机制
- **矛盾与更新处理**：人类习惯或项目技术栈会演变。引擎底层能识别到“时序上的矛盾”（Temporal Changes / Contradictions），利用最新获取的事实去覆盖、更新冲突的旧观点。
- **主动遗忘机制**：提供 `forget` 工具供 AI 主动剪除过期不再有效的信息，防止持久积累下填满全部 Context Window，引发灾难性遗忘。

## 4. 深度辨析：为什么强调 Memory 与 RAG 的核心区别？

Supermemory（及同类前沿项目如 Mem0）极力主张**“不能用 RAG 的方式做记忆”**，原因是两者处理信息生命周期和业务痛点的逻辑完全不同：

| 维度 | 传统 RAG (检索增强) | Memory (智能记忆) |
| :--- | :--- | :--- |
| **核心定位** | 外挂知识库（大模型的移动硬盘） | 用户心理模型与上下文（大模型的情商与习惯） |
| **处理单元** | 粗暴切分的文本块 (Chunks)，含大量噪音 | LLM 提炼后的高价值事实 (Facts)，信噪比极高 |
| **冲突处理** | 无法处理。同时返回互相矛盾的内容导致幻觉 | 基于时间线动态更新，用新偏好覆盖旧偏好（CRUD） |
| **干预方式** | 被动触发搜索（User Trigger-based） | 主动打包成用户画像（Profile）自动注入 System Prompt |
| **生命周期** | 长期留存，多为只读追加 (Append-only) | 支持增删改查及“主动遗忘 (Forget)”时效性信息 |

因此，Supermemory 采用了**混合架构（Hybrid）**：针对长文档和代码集，依然执行 RAG 机制；而针对用户的日常对话交流，则走高密度的 Memory 事实链路。它将两者封装进了统一的 API 中，让 AI 既具备广博的“知识”，又极其懂你的“心思”。

## 5. 商业模式与付费墙设计

虽然项目具备开源属性，Supermemory 针对 SaaS 平台服务设立了严格的付费墙（Paywall）策略，核心通过限制“第三方连接器”来做商业变现：

### 5.1 Free Plan（免费体验版）
- **核心限制**：无法使用 Connections。你只能通过在聊天框内纯手工高频打字，让模型去记录你的喜好（纯对话驱动）。
- **额度限制**：提供基础 100K/1M Tokens 和 10K 搜索查询额度。

### 5.2 Pro Plan（专业版：$19~$20 / 月）
这是解锁该项目“完全体”的形态。
- **解锁 Connections**：支持最多绑定 10 个第三方数据源（如 Google Drive，Notion，OneDrive 等），允许后端通过 Webhook 实时同步知识库作为你的长线背景增强。
- **额度与插件提升**：提供高额查询次数、3M Tokens 缓存并解锁额外的平台插件应用。

因此，如果个人开发者仅用来“跨会话连载编码习惯”，免费版功能就能胜任；若想接入公司和个人飞书/Notion文档，让 AI 懂你整个工作区的沉淀资产，目前该平台的核心卡点在 $19 每月的 Pro 版本上。

## 6. 快速接入体验
通过支持的 MCP 协议，仅需一行命令就能为你现有的 AI IDE/桌面端安装 Supermemory 服务：
```bash
npx -y install-mcp@latest https://mcp.supermemory.ai/mcp --client claude --oauth=yes
```
*(注：可将命令中的 `claude` 替换为 `cursor`, `windsurf`, `vscode` 等对应客户端)*
