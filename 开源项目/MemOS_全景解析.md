# MemOS 全景解析

## 项目简介
[MemOS](https://github.com/MemTensor/MemOS) (Memory Operating System) 是专为大语言模型（LLM）和 AI Agent 打造的一款**内存操作系统（Memory OS）**。它致力于为 AI 系统解决“长期记忆”和“知识管理”的痛点。

## 核心特性
1. **统一的记忆管理 API**：提供统一的接口来实现记忆的存储、检索、编辑和删除，将记忆结构化为图（Graph），拒绝黑盒般的单纯向量存储，使得记忆具备可检查和可编辑性。
2. **多模态记忆支持**：除了常规文本，原生支持图片、工具调用痕迹（Tool traces）以及人设（Personas）的关联检索与联合推理。
3. **多维度的多知识库管理（MemCube）**：支持将多个知识库作为可组合的“记忆魔方”（MemCube）进行管理，能够在不同的用户、项目、Agent 之间实现数据隔离、受控共享和灵活的动态组合。
4. **记忆纠错与反馈（Feedback & Correction）**：允许使用自然语言对现存的记忆进行反馈，随时间推移可以不断纠正、补充或直接替换 Agent 的记忆。
5. **异步吞吐与调度（MemScheduler）**：支持记忆在毫秒级延迟下进行异步读写和调度，保障了高并发生产环境的系统稳定性。
6. **技能记忆与复用（Skill memory）**：让 AI Agent 能够拥有持久的“技能记忆”，不仅不同任务间可以复用，技能还能够自我迭代与进化。

## 应用场景
- **复杂 AI Agent 研发**：充当 AI 的“大脑海马体”，为大规模多步推理的 Agent 提供强有力的持续上下文环境。
- **个性化 AI 助手**：能够精准记录并应用用户的偏好、历史交互，提供更深度的定制化服务。
- **多代理群组协作（Multi-agent collaboration）**：支持跨多代理环境共享记忆网络，避免重复传递上下文，如 OpenClaw 框架中的云端托管和本地 SQLite 离线版插件集成。

## 技术实现与架构
MemOS 的底层实现融合了传统数据库与现代 AI 检索引擎的技术，主要由以下核心模块和技术栈构成：

1. **核心语言与框架**
   - 后端主要基于 **Python** (`FastAPI` + `uvicorn` 作为 REST API 服务器)。
   - 周边生态及插件（如 OpenClaw 插件、看板等）大量使用了 **TypeScript**。

2. **混合存储架构 (Hybrid Storage)**
   - **图数据库 (Neo4j)**：将记忆和实体之间的关系抽象为图结构（Graph），这是其能够被“检查”与“精确编辑”的核心底层支撑，而非传统的黑盒 Embedding。
   - **向量数据库 (Qdrant)**：负责进行语义层面的向量相似度检索。
   - **本地轻量级存储**：在纯本地版插件中，采用 **SQLite** 持久化存储，并结合 **FTS5**（全文搜索）和本地 Vector Search，实现零外部依赖的离线记忆能力。

3. **异步调度器 (MemScheduler)**
   - 采用 **Redis Streams** 作为底层调度与队列机制。
   - 通过将记忆写入、解析、纠错等重度 IO 操作进行异步处理，将吞吐推至更高，保证了毫秒级的响应延迟，解决高并发下记忆操作阻塞 Agent 推理流程的问题。

4. **记忆隔离与组合逻辑 (MemCube)**
   - 在逻辑层面引入了 `user_id` 和 `mem_cube_id`（记忆魔方 ID）。
   - 将各个知识库、不同 Agent 或不同项目的数据封装隔离。在调用检索或写入时，Agent 可以像拼乐高一样挂载不同的 MemCube，以受控的方式进行多方记忆共享。

5. **生命周期拦截 (基于插件)**
   - 以其集成的 OpenClaw Plugin 为例，它是利用框架支持的“生命周期钩子”（Lifecycle hooks），在 Agent 启动前**自动召回（Recall）上下文**并注入 Prompt，同时在 Agent 运行结束后，对对话历史进行**自动摘要与长记忆写入**，大幅降低（高达 72%）每次交互由于携带长程全量历史而产生的 Token 消耗。
