# 如何构建结构化的 Vibe 研发管线工作流

Vibe Coding 应基于标准化的 Agentic Pipeline 推进工作流，用高约束流程替代自然语言闲散对话。

### 1. 战前需求审问与约束期 (Pre-Coding Verification)
禁止在需求未锁定技术实施边界前直接令 AI 生成业务代码：
1.  **意图剖析探查**：审视需求底层旨在解决的问题性质（是数据呈现逻辑缺失，还是底层旧接口缓慢导致）。
2.  **反向逼问机制 (Reverse Prompting)**：面对宽泛需求必须提出要求：“基于此类订单退款逻辑构建，请不要输出任何代码，先反向向我提出三连倒逼提问，补充所有防呆处理和状态机异常边界。验证通过前禁写逻辑！”
3.  **锁定蓝图契约文档 (Contract-Driven Blueprint)**：在双方核定所有硬性盲障后，输出纯技术型接口定义、文件大盘关系拆分和 TypeScript 断言约束，构成锁死契约。

### 2. 强制独立前置调研 (Deep Research)
在正式下达编码指令前，必须要求 AI 建立纯净上下文理解体系。
*   **出具调研断言**：强令模型预先输出 `research.md`，深度详述其对于现有代码的缓存策略、ORM 约定或状态库理解，确保其未发生幻觉方可发包。

### 3. 人机异步批注流 (The Annotation Cycle & Async HITL)
隔离“规划”与“执行”，将人类升格为高并发调度节点：
*   **强制挂起与图纸批注**：命 AI 先出具 `plan.md` 且尾部悬挂“暂不执行”锁。由人类对图中核心组件加注评审意见后再予以放行。
*   **高压线打断机制 (Suspend)**：在初始指令中设定硬拦截网（如涉及数据库变更、鉴权调整、全库依赖变动，立刻中止全自动流并抛出【待审批方案】）。人类利用切频时间处理放行或驳回。

### 4. 模块级强压推进工作流
契约生成完备且落定后，切入执行组装：
*   **解耦化极小模块拆装 (Micro-Batch Delivery)**：面对大粒度复杂应用，将外围功能隔离。指令生成仅具备极高闭环特质的单体逻辑（如强并发锁模块），需附带完整的阻断单测网。
*   **即时阻断核验检查 (Fail-Fast Gatekeeping)**：该封闭组件一旦产出立刻执行边界极值全量核验。发生深度死锁循环及指针越界情况时，要求开发者切断接管干预冲动，全量指认堆栈将原始任务打回系统全盘推翻重制。

### 3. 代码审查及全局知识下沉 (Post-Coding Consolidation)
编译通过并非代码真正落定，必须处理技术沉淀：
*   **双边角色代码审查会战**：触发特构的安全审核 Agent 和注重资源耗损极简架构的 Agent 两段防线。依据原始产出暴露的微小漏缺发起严格的对抗与修正。
*   **全局知识提纯晶化 (Knowledge Harvesting)**：指令系统提取这轮开发中修复过的冗长错误和极其复杂的死结关联，将其降维归纳成极短的前置开发法则真言，永久更新入该系统的说明锚点文件中。

### 4. 强制流式节点管理 (Streaming Management)
防备开发失控的核心机制在于短周期验证与同步：
*   **强制文档映射同频 (Document Synching)**：每完成一个核心模块，必须指令 AI 同步更新对应 `README.md` 或设计图接口。严格锚定“代码与基建文档并轨迭代”。
*   **拦截归档网 (Commit/Backup Gate)**：在放权让模型执行高风险架构变动（大篇幅重构/依赖项升维）前，必须强制打断其推演动作，进行手动或自动的 Git 提交锁结，确保坠机后具备瞬时恢复能力。
