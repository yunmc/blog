# 如何为 AI Agent 团队搭建经验基因库 (GEP)

当多个 AI Agent 或跨独立会话 (Session) 协助开发时，为防止新启动的 Agent 重复排查以往已解决的隐蔽 Bug，需建立结构化的经验基因库（GEP）。

### 1. 知识积累前置至初始编写阶段
经验基因库绝非项目末期的复盘产物，必须在写下第一行代码首日便启动：
*   **起始基建锚定**：在项目初始化阶段，即把领域特殊规则、极其核心的鉴权架构或数据库防腐契约固化入库。
*   **并行收割 (Continuous Harvesting)**：在开发过程中遭遇环境阻断并成功跑通后，不要急于推演下一步。第一动作是立刻中止开发流，强制 AI 提取该经验结晶并注入知识库。

### 2. 构建高结构化知识格式以供精准匹配
未经严谨结构化的自然语段无法被新 Agent 在庞大的上下文中准确命中检索。经验必须是利于多维匹配的机器可读卡片：
*   **多维命中游标**：强制拆分前置索引条件（如 `[技术框架: Flutter]`, `[业务场景: 状态机并发]`, `[报错特征: ensure_ascii]`）。
*   **指令级模板 (Signal-Step-Validation)**：内容必须严格提纯为：捕捉触发信号 -> 1-2-3 极其限定排雷执行动作流 -> 用以复检自省的底层探活指令。严禁注入人工解释性发散废话。

### 3. 在底层配置确立 SOP 读取级最高优先权
在仓库全局环境（如 `.cursorrules` / System Prompt 中），锁定其面对特例环境的高权读取反馈链路：
*   **环境锁阻断机制**：明文声明“一旦捕捉到依赖编译异常或特定环境乱码，严禁基于现有报错胡乱发散盲猜。第一动作必须强制挂起自身操作，静默扫描检索 `./docs/gep/` 经验库，匹配过往对应 Gene 架构并严格复刻脱困！”

### 4. 经验沉淀的思维梯度建设 (四级知识分层)
经验萃取必须遵循由宏观到微观的严格分层，避免上下文污染与知识越权：
*   **企业级 (Enterprise)**：跨部门全局底座。沉淀编码规范、安全红线、鉴权架构、跨域通信契约与大厂合规审查红线。
*   **团队级 (Team)**：特定技术栈协作准星。沉淀特定语言框架（如 Flutter/NestJS）的最佳实践、状态管理流派、代码评审 (CR) 校验清单。
*   **项目级 (Project)**：业务领域隔离区。沉淀当前项目的私域业务逻辑、特定三方接口容错机制、数据字典与特有核心流程图。
*   **个人级 (Individual)**：开发者专属微操域。沉淀个人微调的 Snippets、专属 Prompt 起手式与效率指令别名。
