# 如何写出更高阶的提示词 (Prompt Engineering)

提示词是约束 AI 生成范式的结构化军令状。需通过版式与语意隔离，管理大语言模型的上下文关注盲区（Lost-in-the-Middle）。

### 1. 三明治排版法则 (Sandwich Prompting)
有效规范掺杂超长代码片或海量日志的复合意图防止核心诉求失焦：
*   **顶层定调（顶核约束）**：首行即声明全案定调与该轮修改操作绝对不可碰触的红线范围（如“前端专家权限；任务仅涵盖登录组件补充验证码逻辑；锁死其他一切业务域文件”）。
*   **夹核中段（长数据基带）**：贴入所需的杂长依赖文档、多段报错控制台堆栈记录、核心的待重构代码原版片段。
*   **底置重申（尾核阻断）**：于输入框极末尾端，强制重新输出主核心命令进行拦截对齐。由于其紧靠生成指针，能拉回系统注意力（“再次进行重申比对：所有修改只准围绕表单层发生，绝不允许对数据库原脚本库做连带删除及修正”）。

### 2. 由白话口语转向架构定调传输
*   **❌ 低效松散指令**：“把这个框改成如果密码错了就标红报错且增加一个提醒动画。”
*   **✅ 严谨标准契约**：“模块：登录校验器。1. 接收入参强校验并转换为 `{password: string}` 格式实体。2. 构建正则防线 `/^(?=.*[A-Za-z])(?=.*\d)[A-Za-z\d]{8,}$/` 过滤。3. 若触碰异常需直接外接调取系统级的 `<GlobalAlert>` 组件做抛出回调。”

### 3. 指令层级的范例优先模式 (Few-Shot Demonstration)
*   遇见具备杂乱字符清洗及复合对象层级转换等硬核业务，切忌长篇大论强硬描述原理解释。
*   投喂 2-3 套包含最恶劣环境因子的 `脏数据输入 -> 期望的极其干净的组装输出` JSON 对象比对参考。约束 AI：“免除解释！提取以上三对清洗规律动线模板，直接实现映射函数接口”。

### 4. 导入反向倒逼安全拦截层 (Reverse Prompting)
*   **执行前排雷机制**：发布重大复杂且模糊的需求末尾，必须连带反向要挟卡点：“在查阅此高危退款模块并进行生成前，立刻向我反问三个关于我这个连带模块可能会导致的超卖泄漏暗雷卡点。若没通过该三连拷问，禁止进入实操敲代码步骤！”

### 5. 顶配专家角色赋予 (Persona Prompting)
*   **强制拉升能力下限**：切忌使用“你帮我写一个”等平庸字眼。必须在对话起始通过极其硬核的专家身份约束模型的知识调取域（如：“作为极其苛刻且严格遵守 SOLID 原则的安全架构师”）。以此激发 AI 倾向于主动防御异常的系统潜意识。

### 6. 微步迭代与死局重置 (Iterative & Reset) 
*   **小范围试探拆解**：针对庞大模块，必须拆解为微步推演：构思 -> 局部防线设定 -> 生成 -> 审查 -> 下一轮进阶。
*   **坚决截断防沉没 (Git Revert)**：一旦发现 AI 陷入试图通过“增加恶心补丁代码”去修复报错的怪圈时。立刻执行 Git 回滚代码并废弃当前对话，重新精简 Prompt 方向，绝不在已污染的基础上继续堆砌。

### 7. 驱动 AI 进行提示词自循环优化 (Case-Driven Prompt Optimization)
严禁人工闭门凭空猜想和调试 Prompt，必须引入真实测试范本进行高精度的机器重构与自我提纯：
*   **双向样本对冲 (Bad/Good Case)**：收集开发中 AI 生成失败的废弃代码 (Bad Case) 以及你人工修正后的标准实现 (Good Case)，将两者并列投喂给当前同代标杆级最强模型（如 Claude 3.5 Sonnet / GPT-4o 等顶级闭源大模型）。
*   **逆向推演与重构**：强迫最强 AI 推演原本 Prompt 导致 Bad Case 的认知裂缝，总结核心缺陷，并让其直接输出一套已打过补丁的最新高阶 Prompt，实现经验沉淀与降维打击。
