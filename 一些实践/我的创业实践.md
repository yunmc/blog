# 空降产研团队Leader的最重要三件事

空降成为产研团队（产品+研发）的 Leader 是一项极具挑战的任务，最大的忌讳就是“新官上任三把火”盲目改革。在入职的前 30-90 天，最应该聚焦于以下三件核心事项：

## 1. 深度倾听与摸底，建立基本信任（看人与文化）
空降 Leader 面临的最大危机是团队的防御心理。首要任务是了解现状并安抚人心，而不是立刻下达指令。
* **全员 1-on-1：** 与团队中的每一个人（产品经理、开发、测试、设计）进行一对一沟通。了解他们的职责、遇到的痛点、对当前流程的看法，以及他们的职业诉求。
* **识别核心骨干与非正式领袖：** 弄清楚谁是技术骨干，谁是团队的“意见领袖”。争取他们的支持是推行任何后续计划的基础。
* **只听不改：** 在前两周，重点是观察团队的日常运作机制（如站会、评审会、发布日），记录问题但**不要轻易做出评判或立刻推翻现有流程**。

## 2. 梳理业务现状与研发交付链路（看事与流程）
作为产研 Leader，必须对团队交付的“产出物”和“交付过程”有绝对清晰的掌控。
* **摸清业务切面与技术负债：** 深入了解当前产品的核心业务指标（OKR/KPI）是什么？线上最大的痛点是什么（客诉密集点、性能瓶颈）？系统架构存在哪些历史包袱和高风险点？
* **诊断交付效能：** 从需求池（Backlog）到 PRD 评审，再到开发、测试和上线，走查整个运转流程。找出效率最低的一环：是产品需求不清晰导致开发返工？还是测试环境不稳定导致阻塞？或者是发布流程过于繁琐？
* **向上对齐期望：** 必须与上级（老板或业务线负责人）深度沟通，明确他们对团队在未来 3-6 个月的最核心期望是什么（是提升交付速度？降低 Bug 率？还是攻克某个重大业务难关？）。

## 3. 选择痛点，拿下一个高质量的 "Quick Win"（建威信与指方向）
信任不仅来自于沟通，更来自于带来的实际价值。需要通过一场小战役的胜利来证明自己。
* **寻找高ROI的突破口：** 结合前两步的摸底，找出一个**大家都觉得痛、但解决起来难度可控**的问题。比如：优化一个极其耗时的编译过程、统一并规范化原本混乱的 API 文档、或是砍掉几个明显无价值的边缘需求以减轻团队当下的加班压力。
* **带领团队拿到结果：** 亲自推进这个改进，消除阻塞点。当团队真切感受到“新来的 Leader 帮我们解决了实际问题”，威权和信任度就会迅速建立。
* **基于 Quick Win 制定中短期规划：** 借着初步建立的信任，再逐步引入需要推行的机制改革和接下来的产品/技术演进方向。

**总结：先懂人、再懂事、最后带大家打个小胜仗。** 保持谦逊和同理心，把团队的成功当作站稳脚跟的最强武器。
