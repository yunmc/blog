# AIGC 公务员命题系统全景解析说明文档

## 一、 材料获取及处理 (数据流水线)

**本阶段核心目标**：将未经处理的杂乱原始统计数据，经过标准化清洗、切片、优化、重构，转化为长度在500字左右、信息自包含、具有公考“真题味”的高质量多模态材料切片，为下游命题提供标准弹药。

### 1. 原始数据获取 (待交接模块)
*   **材料爬取**：定向抓取国家或地方统计局的国民经济运行情况、公报等权威文字与表格数据。
*   **图片 OCR**：将扫描版或纯图片版公报转化为可编辑数字文本。

### 2. 材料文本清洗 (纯代码/规则驱动)
*   **标签游离与标记**：提取并单独存放 `<img>` 和 `<table>` 标签，清除 `table` 内部无关标签（如 `p`、`span`）。
*   **排版净化**：强制清除内容中其他所有的 HTML 标签（`<html>`、`<body>`、`<strong>`、`<br>` 等）。
*   **特征占位**：在原文图表位置留下“图x”、“表x”的标准化占位符。
*   **字符清洗**：利用正则或代码清理多余特殊符号（如 `\t`、空格、`&amp;`、`&nbsp;`）。

### 3. 基于规则的材料压缩
*   **尾部降噪**：精准探测并移除材料末尾的解释性文本（如“附注”及其后续内容、“注①-⑩”等），避免无用数据进入命题库。

### 4. 基于 LLM 的材料拆分 (核心结构化)
*   **语义切片**：按照逻辑语意将长篇幅公报切分为多个独立段落（尽量不破坏完整主体，字数不满800字尽量不拆）。
*   **实体连通性扫描（防冲突机制）**：在强行拆断前，检查该段文字内是否有同时提及了“图1”和“表2”中的变量，如果有，将它们作为“不可分割原子图块”合并输出，避免生成的题目在查表时无据可依。
*   **多模态绑定**：扫描段落中的“图x”、“表x”占位符，将步骤2中提取的表格与OCR数据精准挂载到当前切片。
*   **信息自包含（Self-contained）**：LLM 强制补全切分后段落缺失的时间、位置等上下文要素，确保切片能够独立成文。
*   **兜底质检（数据防丢）**：比对原始全文与所有拆分段落的数据总和。发现统计数据遗漏，少量则塞回对应切片，过多则触发重拆。

### 5. 基于 LLM 的文本优化 (公考特征化)
*   **脱敏与泛化**：将具体地名替换为“X市/X省”。
*   **格式自洽**：去除因为拆分导致的孤立序号（如“二、”、“三、”）和无意义的引导说明语（如“发布时间：xxx”）。
*   **优化复检**：再次核验是否因为改写导致了核心统计数据的篡改或丢失。

### 6. 基于 LLM 的文本压缩 (精细化定型)
*   **去冗缩写**：针对拆分后仍然超过500字左右的臃肿切片，指挥 LLM 删减不含统计核心数字的冗长文字描述，进行瘦身定型。
*   **压缩复检**：确保数据事实 100% 完整保留。

### 7. 文字转图片 (多模态渲染)
*   **表现力丰富**：选取部分适合图表化的数字文本，利用代码或绘图库将其反向渲染为柱状图、折形图、饼图等，形成高质量的“图文混合”材料。

### 8. 材料打标 (命题路由前置)
*   **标签化**：根据切片内包含的数据特征（是否有基期、增速、比重、部分与整体关系），给该材料打上“适合题型”的标签（如：适合出“基期量计算题”、“综合分析题”），为下游流水线提供调度依据。

## 二、 命题阶段 (基于材料的自动化出题流水线)

**本阶段核心目标**：针对前置处理好的结构化材料切片，利用大模型（Agentic Workflow）的多轮对话、交叉验证能力，自动化生产包含题干、选项、正确答案及高质量解析的“基础题”。

### 1. 基础题命题 (受控生成引擎与 Few-shot 模板)
*   **动态组卷调度与知识点约束 (引擎大类控制)**：一份标准的资料分析题组通常由5道题（4道基础题+1道综合题）构成。整个命题过程不是大模型盲目发挥，而是由**题型抽排调度引擎**被动强控完成的：
    1. **大类防撞车**：强控同一篇材料抽取的这5道题，必须跨越不同的“一级题型”大类（如不能同时出现3道都是基期量的计算题），杜绝考点极度雷同。
    2. **调度优先级与频次控制**：系统有一套完整的逐级降频抽排逻辑（优先抽取未出过且难度符合的题型 -> 其次是已出过但适应的），并在后台支持“标红冷门题少出（如控制10题只出现1题）”、“标绿重点题多出”的人工干预。
    3. **精准派发**：最后结合材料前置的特征标签（如包含现期量和增速），下达具体的题目生成指令。
*   **弃用复杂 RAG，转用静态 Few-shot 路由**：由于资料分析基础题型高度收敛（仅20种左右核心题型），系统**不使用**重成本的向量检索 RAG，而是自建了一个“静态 Few-shot 模板库”。当前端下发具体题型指令时，系统根据 Key 提取对应的多道优质真题模板，作为 Context 随材料一并注入，从而强制大模型模仿公考设问话术。
*   **题干与答案生成**：由于不引入外部代码引擎计算，大模型（相当于出题人角色）利用其内部逻辑能力，根据指定的考点和对应数据推理、演算出正确的题干设问，并得出它认为的“唯一正确答案”。

### 2. 题干质检与答案交叉核验 (双轨查错)
*   **基础题题干质检**：初步检查生成的题干是否存在表述不清、存在多主语歧义、无明确时间落点等公考常识性错误。
*   **检查选项是否正确（交叉核对）**：系统引入**另外一个独立的 LLM（即验证 Agent）** 扮演考生的角色。把刚才生成的材料、新题干喂给这个 Agent，让它独立作答。如果验证 Agent 计算出的答案与出题 Agent 给出的目标完全不一致，系统直接熔断拦截，废弃该次生题，防范数值幻觉。

### 3. 【核心骨架】选项设计与质检 (强规则引擎提升“真题味”)
*   **基于 Rules 的定向干扰选项设计**：
    * 这一步是决定题目生死的关键（决定了这道题是否具备“真题感”的陷阱属性）。
    * 系统**摒弃了不稳定的开放式启发式生成，转而采用“强规则引擎（Rules Engine）”**。在 Prompt 设计上，直接通过硬编码写死每一种题型的错构策略，逼迫大模型严格当作“填空机器”执行。
    * **常见干扰项策略设计（Rules）示例包含**：
        1. **Rule 1 - 粗心陷阱**：例如求基期量，强制要求将已知的现期量作为某个错误选项。
        2. **Rule 2 - 概念偷换陷阱**：例如题干考查增长“量”，强制要求选项用增长“率”计算得出一个值。
        3. **Rule 3 - 符号法则错误陷阱**：强制进行“增降”、“百分点和百分号（%）”方向反转的计算。
*   **选项设计质检**：检查生成的 A、B、C、D 四个选项是否具有区分度并满足实战考培规范。主要校验包括：
    1. **数值区分度硬控**：严禁数值类选项过度接近（例如 A.99.7、B.99.8 是绝对不允许的）；同时**强制要求四个选项的首两位数字不能全部相同**，为后续应用“截位直除法”等速算技巧留足排他空间。
    2. **时空要素完整度**：生成的选项中必须严格包含与其对应的年份、时间、地点等主语信息，防止出现歧义。
    3. **防虚假计算项**：绝对不允许出材料中未提供数据的“无解错误选项”，除非该选项文本直接表述为“根据已知数据无法计算”。
    4. **防双选与送分**：明确答案唯一性；各选项须相互排斥，且筛除严重违反常识直觉的“傻瓜送分项”。

### 4. 终极闭环与解析生成
*   **全链作答验证 (盲测主动返空机制)**：将完整题干及 A、B、C、D 四个最终选项发送给终极验证大模型“闭卷做题”。针对大模型为了完成任务经常会**“瞎编条件/面对无解题目硬猜答案”**的痛点，系统在此处特设了**“允许返空”的安全阀**：明确告诉验证 Agent，如果发现四个选项基于材料都算不出来，必须坚决杜绝“胡编乱选”，直接返回“无正确答案（返空）”。系统一旦捕捉到此防幻觉信号，立刻熔断该题并打回重造。
*   **题目解析增强与格式化**：
    * 大模型被要求输出不仅带有最终运算结果，同时包含“公考特制逻辑”的详细解析。系统在解析生成时设定了严格的**解答方法优先级**：**优先使用针对题型的速算技巧/口诀（如两期比重比较口诀） > 次选通用技巧（如截位直除、排除法、尾数法等） > 最后才允许带入基准公式死算**，确保生成的解析具有真正的“名师辅导价值”。
    * 将解析结果执行强规则校验和数据结构固定格式化（保障入库），并进行**解析质检**，剔除掉逻辑跳跃甚至包含“因为此选项就是答案，故选此”等形式化无用的话术。

### 5. 人工把控与宏面打分
*   **质量打分/盲评**：经过所有机器人自动化闭环跑通出库的题目。在入库前最终还需要一环——交给高阶的人类审核员（盲评）或者是最高算力大模型（如 GPT-4/Claude 3 Opus）打下最终决定题库权重的“星级（如难度系数贴合度评比）”。

## 三、 综合题命题阶段 (跨考点复杂逻辑流水线)

**本阶段核心目标**：针对资料分析题中难度最高、综合性最强的“综合分析题”（如：“以下能够从所给材料中推出的是？”、“以下关于各产业发展的说法，正确/错误的是？”），利用大模型生成跨段落、跨考点且信息无极度重合的四个独立陈述选项，并完成严密的逻辑回溯验证。

### 1. 综合题命题 (多维考点聚合构建)
与基础题（一题一问）不同，综合题的命题本质上是“并行生成四道隐藏小题”作为选项。
*   **历史题干黑名单机制（干扰规避）**：在综合题的命题入参（Prompt Context）里，加入该篇材料已经出过的前 4 道【基础题】的考点 Tag（如已考过“现期比重”）。严令禁止在【综合题】的选项中生成与黑名单高度重合的考点，保证综合题的“新颖性和拓荒感”，避免公考中的“漏题撞车”。
*   **多维考点抓取**：大模型需要阅读完整的（或多个）材料切片，同时提取出四个不同的数据或逻辑维度（例如：A项考直接查找、B项考比重变化趋势、C项考基期量计算、D项考总体增速判断）。
*   **陈述句生成与正误设定**：系统指令大模型针对这些考点分别生成相应的判断性陈述句。此时需强制在 Prompt 中设定“一正三误”（选是题）或“三正一误”（选非题）的刚性模板，从而一次性炮制出完整的 A/B/C/D 四个选项。
*   **强规则设计(Rules)延续**：在错项的设计上，同样沿用规则引擎，如“年份偷换”、“总体与部分偷换”、“绝对化表述（如‘均大于’取代‘绝大多数’）”等专属公考的错构陷阱。

### 2. 综合题选项验证 (高危易错层单独拦截)
综合题因为同时包含四个计算逻辑，任何一项出现“计算幻觉”都会导致整题作废（出现多选或无答案）。
*   **切片式拆解验证**：由于选项逻辑互不干涉，系统将这道综合题“拆分”成四个独立命题。
*   **四线并发交叉核验**：派出独立的验证 Agent，拿着原文材料，对这 A、B、C、D 四个选项进行并发式“证伪”与“证实”。如果验证 Agent 判断出的答案正确数量与出题设定的规则（例如仅有一个选项对）不符，那么该综合题强制执行退回与局部重构。

### 3. 制定题目解析与格式化解析 (四核解析模式)
*   **独立制定选项解析**：在综合题中，四个选项必须拥有完全独立的解析段落，不能含糊带过。系统要求大模型逐一对 A、B、C、D 选项输出“定位原文片段 + 计算公式（或定性判断依据） + 正误结论”。
*   **综合题格式化解析**：利用代码层面的强规则校验，要求必须输出标准的诸如“A项：定位材料第一段...，计算可知..., **错误**。 B项：定位表格二...，**正确**。”的格式，确保数据结构的一致性以便入库。

### 4. 综合题解析质检与综合评分 (终局封口)
*   **综合题解析质检**：针对生成的大段解析做再一次核验。重点检查解析中引用的数据原值是否与材料切片出现矛盾（防止大模型在解析解释阶段发生二次幻觉）。
*   **质量打分/人工盲评**：经过层层自动化冲刷后，最后这道极其难生成的“压轴综合分析题”将被送往高阶打分节点（由 GPT-4/Claude 或真人专家打分）。根据其综合难度、选项的彼此排斥度、以及陷阱设计的高阶感进行验收评比，随后完成出卷封装。
