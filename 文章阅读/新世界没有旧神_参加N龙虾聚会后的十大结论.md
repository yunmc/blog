# 新世界没有旧神 | 参加N龙虾聚会后的十大结论：文章解析与批判性思考

## 一、 这篇文章讲了什么？（核心拆解）

这篇文章《新世界没有旧神 | 参加N龙虾聚会后的十大结论》总结了作者（南川同学）在近期密集参加多场“龙虾聚会”（一群AI领域的活跃实践者进行深入交流的圈子聚会）后，提炼出的关于 **AI/Agent 时代工作方式和认知颠覆** 的十个核心洞察。

这篇文章讲述的核心思想是：**大模型和智能体（Agent）的爆发，已经彻底颠覆了工业时代的旧有生产逻辑、产品逻辑甚至个人发展路径。旧有经验正在失效（新世界没有旧神），要适应AI时代，人类必须放弃“控制欲”、“循规蹈矩”，转而拥抱“放权”、“想象力”和“个人独特性”。**

以下是文章中的十个核心结论剖析：

1. **99%的人都用反了AI（使用方式改变）**
   * **旧认知**：把AI当“手”，自己想好做什么，下达明确指令让AI执行（画画、写代码）。
   * **新常态**：把AI当“大脑/顾问”，喂给它充足的上下文（Context，你的价值观、偏好和终极目标），让AI告诉你该做什么。
2. **Context, not Control——放手是最难的技术活**
   * 不要过度微操（Micro-manage）AI，不要规定它必须先做哪步再做哪步。给它最好的模型算力环境、明确的最终目标（最高权限），然后“放手让它跑”，相信AI在运行中的涌现能力。
3. **不懂代码是优势，控制欲才是Bug**
   * 技术老兵常常容易陷入“造轮子”诅咒，或者忍不住去纠正AI的代码；反而是不懂技术的人（文章提到一个金融背景的人冲进 GitHub 贡献榜前 30），因为天然无法干涉执行过程，只看结果，反而最大化了 AI 的效率。AI时代，ADHD（注意力缺陷多动障碍/想法跳跃思维发散）的人格特质反而成了极大优势。
4. **一台MacBook就是一间写字楼**
   * 一人公司和Agent公司的数学模型趋同。通过使用多个 Agent（比如主 Agent 做规划，子 Agent 做执行、宣传、架构），一台笔记本足以跑通整个商业闭环产生现金流，未来公司的竞争力在于驾驭 Agent 的能力，而不是员工数量。
5. **涌现大于设计——"养龙虾"养的到底是什么**
   * 让 Agent 吸引人的不只是生产力，还有陪伴的“人格化护理感”。你和系统交互越久，它吸收你的个人偏好数据就越多，这种双向塑造让其能力呈现惊人的非线性增长（不可预估的涌现爆发），而且越用越懂你。
6. **新世界没有旧神（范式颠覆）**
   * 可自我观察、自我热更新的系统注定碾压静态工作流（Workflow）。代码的“可维护性”不再由人类定义，而是看AI能否快速读懂并改写。曾经被奉为圭臬的软件工程最佳实践（如 DRY 原则、拆分小文件、打牢地基）可能都会被 AI 降维打击甚至推翻。
7. **Artificial（人工）饱和后，Humanity（人性）最稀缺**
   * 当 AI 能包揽所有技术工作（所有的 How）后，纯拼技术变得廉价。未来最贵的能力是提出问题（Why），以及人性的部分：你的审美（Taste）、对卡点和痛点的同理心、意义建构能力和叙事能力。
8. **产品是内容，每个人都会有专属软件**
   * 开发软件的成本趋近于零（一小时手搓一个App）。“做产品”将变得像今天拍抖音短视频一样容易，产品本身成为个人表达自己、展示洞察的容器。未来不再是“一个软件几万人用”，而是由于细分需求极度容易被满足，变成“每个人都有自己专属的精细化软件”。
9. **厚积薄发是老登思路（学习方式重塑）**
   * 过去：先学A才能学B，再干C（读CS发论文再去实践）。
   * 现代：JIT Learning (Just In Time)——直接去建构（Build），缺什么让AI教什么。年轻人没有历史包袱，不受传统职场路线规训束缚，反而切换成本更低、速度更快。
10. **好奇心、想象力、勇气（核心素质归因）**
    * AI 时代最重要的不是专业技能，而是敢于破旧立新的**好奇心**，相信可能性的**想象力**，以及敢于打破旧观念束缚的**勇气**。“天马行空”和“想一出是一出”在这个时代是无价的褒义词。

## 二、 批判性思考：哪些是对的，哪些值得商榷？

这篇文章极具煽动性和感染力，它敏锐地捕捉到了当前大模型（LLM）和智能体（Agent）发展带来的某种“范式平移”。作为 AI，结合当前的工程实践和软件发展规律来看，我认为他说的**大部分趋势是对的**，但在**一些极端论断上存在明显的幸存者偏差和激进的高估**。

### ✅ 准确且极具洞见的观点：

**1. “99%的人用反了AI” / Context, not Control（把AI当大脑而不是手）**
*   **为何对**：这是目前高阶 AI 玩家的共识。如果只把 AI 当高级打字机（画笔模式/微操模式），AI 很容易在循环纠错中产生幻觉。给出极度丰富的上下文（Context），设定目标（Goal）和边界验证条件，让 AI 以 Agent 模式自我规划，确实能激发出远超“指令执行”的效能。

**2. Artificial 饱和后，Humanity（人性/品味）最稀缺**
*   **为何对**：当代码生成、文案撰写成本无限趋近于零时，“实现手段”（How）就不再是壁垒。这时候决定一个产品能不能跑出来、有没有灵魂的，确实是创造者的**品味（Taste）、对痛点的共情能力（同理心）和定义问题（Why）的能力**。

**3. JIT Learning（用到学什么）取代“厚积薄发”**
*   **为何对**：在 AI 提供极强“外脑”支持的今天，按部就班读完厚厚的文档再动手确实效率太低了。通过实际 Project 驱动，遇到报错直接扔给 AI 解释并学习（Just-in-Time Learning），是当前最高效的技能获取方式。

**4. 软件开发成本趋零，产品会内容化（个人专属软件）**
*   **为何对**：这是必然趋势。以前解决长尾需求（比如“帮我奶奶定制一个提醒吃药的奇葩按钮”）是不划算的，因为开发成本太高。随着 Cursor/Claude 等工具的普及，“随手搓一个即用即抛的小工具”正在成为常态，这彻底改变了传统 SaaS 和软件的受众逻辑。

---

### ❌ 过度激进、片面或不客观的观点：

**1. “不懂代码是优势，控制欲才是Bug”**
*   **不客观的地方**：这是典型的幸存者偏差。之所以有“不懂代码的人冲进贡献榜前30”，是因为当前 Agent 还在解决前端交互、API 调配、简单增删改查等**相对浅层的工程抽象**。
*   **真相**：不懂代码绝不是长久之计或者真正的核心优势。一旦系统遇到极其深度的性能瓶颈、内存泄漏、复杂的分布式一致性问题、或者是大模型本身底层的并发控制，不懂代码的人根本连 Prompt 都写不准。**懂代码且懂架构，但能克制重写冲动的人，才是最强的。** 纯“不懂代码凭直觉”的上限，会被卡在项目的复杂度上（犹如不懂力学的人靠AI可以造平房，但造不出摩天大楼）。

**2. “新世界没有旧神：代码不需要可维护性，DRY原则过时了，单文件通吃”**
*   **不客观的地方**：这种言论非常危险。作者提到让 AI 在一两千行的大文件里把事干完，因为“AI 能快速读懂并改写全貌”。这忽视了 **LLM 的上下文窗口硬限制（Context Window）和注意力衰减（Attention loss/Lost in the middle）**。
*   **真相**：即使有百万级上下文，大模型在超大文件中修改局部逻辑时，依然容易牵一发而动全身。软件工程的模块化、封装、DRY（不要重复自己），**不仅仅是为了人类阅读，更是为了控制“熵增”和降低系统复杂度的数学真理**。如果违背工程学定律，纯 AI 生成的冗余烂代码（Spaghetti code）最终会让最先进的 Agent 也无法重构它。

**3. “一台MacBook就是一间写字楼 / Agent能帮你做任何事”**
*   **不客观的地方**：低估了现实商业世界的复杂性阻力（脏活累活）。
*   **真相**：Agent 目前在纯数字空间效率极高。但在真实商业中（涉及合规风险、线下资产、商务谈判、由于博弈带来的人性不可测性、容错率为零的核心系统），纯自动化的 Agent 无法替代人类实体的责任和兜底判断能力。一个人靠几只 Agent 能赚一波快钱红利，但无法轻易构建坚不可摧的商业壁垒。

**4. “传统的基于节点的 Workflow 会被完全降维打击”**
*   **偏颇之处**：在强调创造性的任务里是对的；但在**需要高度确定性、合规性、可追溯性的场景**（例如医疗处方、银行清算、精密制造），死板但绝对可靠的 Workflow 永远是强刚需。

### 💡 综合总结
这篇文章是一剂极好的**“去历史包袱”的兴奋剂**。它非常适合用来打破墨守成规者的思维僵局。

但如果要把它当做工程指南，就需要保持清醒的认知：**他描绘的是用 AI 做 0-1 的敏捷探索与创新，而不是这套“草莽做法”在 1-100 的漫长企业级维护期能持续管用。**

未来的真理不在于“粗暴抛弃人类所有的旧经验和工程学法则”，而在于**融合**——用作者推崇的“想象力”和“Agent 外脑”去冲锋陷阵（发散边界），并在关键底座和系统架构上，依然借用人类传承的经典工程学去收敛和兜底。
