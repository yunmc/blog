# 自建 AI API 中转平台 — 需求分析

> 分析日期：2026-03-13
> 参考对象：AICodeWith.com 等国内 AI API 中转服务

## 一、核心技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| API 网关 | Nginx / Kong / 自研 Go/Node 反向代理 | 核心组件，负责请求转发、鉴权、限流 |
| 后端服务 | Go / Node.js / Python | 用户管理、计费、API Key 管理 |
| 前端 Dashboard | React / Vue / Next.js | 用户控制台、用量统计、充值 |
| 数据库 | PostgreSQL / MySQL + Redis | 用户数据、积分余额、调用日志 |
| 支付系统 | 微信支付 / 支付宝 SDK | 人民币收款 |

## 二、前置条件

- **官方 API 账号**：Anthropic / OpenAI 的 API 访问权限和充足余额
- **海外服务器**：至少一台海外 VPS（美国/日本/新加坡），用于转发请求到官方 API
- **国内 CDN / 加速节点**：降低国内用户延迟（阿里云/腾讯云海外加速）
- **域名 + 备案**：面向国内用户需要已备案域名

## 三、核心功能开发清单

1. **反向代理层** — 接收用户请求，转发到官方 API，返回结果（最核心，技术难度不高）
2. **鉴权系统** — API Key 生成 / 验证 / 权限管理
3. **计费引擎** — 按 token 用量计费，实时扣减积分
4. **用户系统** — 注册 / 登录 / 个人信息
5. **支付集成** — 微信 / 支付宝充值积分
6. **用量监控** — 调用统计、余额告警
7. **多模型路由** — 根据请求中的 model 字段路由到不同上游 API

## 四、运营层面

- **资金**：需要垫资购买官方 API 额度（用户先充值，你再调用官方）
- **合规**：ICP 经营许可、增值电信业务许可等
- **风控**：防止用户滥用（生成违规内容）导致官方封号
- **客服**：中文技术支持

## 五、风险点

| 风险 | 说明 |
|------|------|
| 官方封号 | Anthropic/OpenAI 可能禁止第三方转售，随时封禁 API Key |
| 合规风险 | AI 内容审核、数据出境、经营许可 |
| 汇率风险 | 人民币收入 vs 美元支出 |
| 竞争激烈 | 门槛低，同类服务多，价格战 |

## 六、最小可行版本 (MVP) 估算

### 技术实现

- 一个 Go/Node 反向代理 + 简单 API Key 验证 + SQLite 记录用量 → **1-2 天**
- 用现成 Admin 模板搭 Dashboard → **2-3 天**
- 支付先手动充值（微信转账），后期再接支付 API

### 基础设施

- 一台海外 VPS（$5-20/月）
- 一个域名

### 结论

技术难度不高，核心就是一个**带鉴权和计费的反向代理**。难点在运营和合规。
